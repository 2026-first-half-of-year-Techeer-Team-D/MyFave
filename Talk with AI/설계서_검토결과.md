# MyFave 프로젝트 설계서 검토 결과

> 검토 대상: `MyFave_프로젝트_설계서.md`
> 검토 기준: 실제 코드 세팅 상태(`backend/`, `docker-compose.yml`, `build.gradle` 등)와 대조

---

## 잘 된 부분

- Java 21 + Spring Boot 3.4.x 조합은 최신이고 적절함
- 도메인 중심 패키지 구조 (auth, user, product, cart, order 등) 명확
- 환경별 설정 파일 분리 (`local` / `prod`) 잘 설계됨
- JWT secret을 `${JWT_SECRET}` 환경변수로 관리 — 보안 측면 좋음
- Docker Compose (PostgreSQL + Redis + App) 이미 구현됨
- `.env.example` 제공 — 협업 시 편리
- `GlobalExceptionHandler` + `CustomException` + `ErrorCode` Enum 구조 — 업계 표준
- `ApiResponse<T>` 공통 응답 구조 — 명확하고 일관성 있음
- Git Flow 브랜치 전략 (main → develop → feature/) 적절

## 문제점 및 수정 필요 사항

### 🔴 높은 우선순위 (필수)

#### 1. `application.yml`의 `ddl-auto: validate` 주의

- 공통 `application.yml`에 `validate`가 설정되어 있으면, 로컬 초기 실행 시 테이블이 없어서 앱이 시작되지 않음
- `application-local.yml`에서 `create-drop`으로 오버라이드하므로 로컬에서는 괜찮으나, 설계서에 이 동작을 명시해야 함
- **더 큰 문제**: `application-prod.yml`의 `ddl-auto: none`이면 최초 운영 배포 시 테이블이 아예 없음 → **Flyway 또는 Liquibase 없이는 운영 불가**

#### 2. DB 마이그레이션 도구 부재

- 운영(`ddl-auto: none`)에서 스키마 관리 방법이 없음
- 팀원이 entity를 변경해도 prod DB에 반영되지 않음
- **Flyway** 추가 권장:
  ```groovy
  implementation 'org.flywaydb:flyway-core'
  implementation 'org.flywaydb:flyway-database-postgresql'
  ```

#### 3. 누락된 의존성: Spring Mail

- `ErrorCode`에 `AUTH_TOO_MANY_REQUESTS`(인증코드 발송 초과), `AUTH_INVALID_VERIFICATION_CODE`, `AUTH_EXPIRED_VERIFICATION_CODE`가 존재
- 이메일 인증을 사용하려면 `build.gradle`에 추가 필요:
  ```groovy
  implementation 'org.springframework.boot:spring-boot-starter-mail'
  ```

#### 4. 누락된 의존성: OAuth2 (소셜 로그인)

- `ErrorCode`에 `AUTH_INVALID_SOCIAL_CODE`, `AUTH_SOCIAL_PROVIDER_ERROR`가 존재 → 소셜 로그인 구현 의도가 명확
- `build.gradle`에 추가 필요:
  ```groovy
  implementation 'org.springframework.boot:spring-boot-starter-oauth2-client'
  ```

#### 5. `global/security/`에 `UserDetailsServiceImpl` 누락

- `JwtAuthenticationFilter`가 사용자 정보를 조회하려면 `UserDetailsService` 구현체가 필요
- 프로젝트 구조에 `UserDetailsServiceImpl.java`가 없음 → 추가 필요

#### 6. `payment` 도메인 처리 방법 미명시

- `ORDER_PAYMENT_FAILED(502, "외부 결제 서비스 오류")` 에러코드가 있음
- 결제를 order 도메인 내에서 처리할지, 별도 `payment` 도메인으로 분리할지 설계가 없음
- PG사 연동 계획(토스페이먼츠, 카카오페이 등)이 전혀 없음 → 방향 결정 필요

### 🟡 중간 우선순위 (권장)

#### 7. 누락된 도메인: `review` (상품 리뷰/후기)

- 쇼핑몰의 핵심 기능인데 도메인이 없음
- `ErrorCode`에도 review 관련 에러코드 없음
- 추가 필요: `domain/review/` + `REVIEW_NOT_FOUND`, `REVIEW_ALREADY_EXISTS` 등

#### 8. 누락된 도메인: `wishlist` (찜 기능)

- 서비스명이 "My **Fave**"인데 즐겨찾기/찜 기능이 없음
- `domain/wishlist/` 추가 고려 필요

#### 9. 공통 클래스 `BaseEntity` 누락

- 모든 도메인 엔티티에서 `createdAt`, `updatedAt` 같은 공통 필드가 필요
- `global/common/BaseEntity.java`에 JPA Auditing 적용 클래스 추가 필요:
  ```java
  @MappedSuperclass
  @EntityListeners(AuditingEntityListener.class)
  public abstract class BaseEntity {
      @CreatedDate
      private LocalDateTime createdAt;
      @LastModifiedDate
      private LocalDateTime updatedAt;
  }
  ```

#### 10. 공통 클래스 `PageResponse` 누락

- 상품 목록, 주문 목록 등 페이지네이션 응답이 필요한데 공통 래퍼 클래스가 없음
- `global/common/PageResponse.java` 추가 필요

#### 11. `ApiResponse`에 `error()` 정적 팩토리 메서드 누락

- `ok()`, `created()`는 있으나 `error()` 없음
- `GlobalExceptionHandler`에서 에러 응답 반환 시 패턴이 불일치할 수 있음

#### 12. 파일 업로드 인프라 없음

- 상품 이미지, 콘텐츠 이미지 업로드를 위한 스토리지 전략이 없음
- AWS S3, Cloudflare R2, 또는 로컬 파일시스템 중 선택하고 설계에 반영 필요

#### 13. `content` 도메인 `ErrorCode` 누락

- `domain/content/`가 있는데 `ErrorCode`에 content 관련 에러 코드가 없음
- `CONTENT_NOT_FOUND` 등 추가 필요

### 🟢 낮은 우선순위 (개선 권장)

#### 14. `influencer.user-id: 1` 하드코딩 설계 문제

- 인플루언서 ID를 1로 고정하는 방식은 확장성 없음
- `User` 엔티티에 `ROLE` 필드(`ADMIN`, `INFLUENCER`, `USER`)를 두는 방식 권장

#### 15. Git 브랜치 전략에 `hotfix` 누락

- 운영 긴급 수정을 위한 `hotfix/` 브랜치 전략이 없음
- `main → hotfix/XXX → main + develop` 흐름 추가 권장

#### 16. API 버전 관리 전략 없음

- `/api/v1/` 형태의 URL 버전 관리 전략 명시 필요
- 최소한 공통 prefix 설정: `server.servlet.context-path: /api/v1`

#### 17. `application-local.yml`의 `create-drop` 주의

- Docker Compose로 PostgreSQL 볼륨을 유지하면서 `create-drop`을 사용하면 앱 재시작 시 매번 데이터가 사라짐
- 개발 초기엔 `create`가 더 적합하고, Flyway 도입 후엔 `validate`로 전환

#### 18. prod 환경에 staging 프로필 없음

- `application-local.yml`, `application-prod.yml`만 있음
- QA/스테이징 환경이 필요하다면 `application-staging.yml` 추가 고려

#### 19. 설계서에 `docker-compose.yml` 언급 없음

- 이미 구현된 `docker-compose.yml`이 설계서에 반영되지 않음 (문서 불일치)

## 의존성 최종 권장 목록 (설계서 대비 추가분)

```groovy
// 이메일 인증 (ErrorCode에 인증코드 관련 에러 있음)
implementation 'org.springframework.boot:spring-boot-starter-mail'

```
