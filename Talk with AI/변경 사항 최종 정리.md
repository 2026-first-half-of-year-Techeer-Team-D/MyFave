| # | 대상 | 변경 내용 |
|---|---|---|
| 1 | ENUM 추가 | `category_code_enum ('ALL', 'TOP', 'BOTTOM', 'OUTER', 'ACCESSORY')` 생성 |

## 4. ERD에서 수정해야 할 사항

> `postgresql.example`은 수정 완료됐지만, ERD 툴(DB 다이어그램)에는 아직 반영되지 않은 항목들

### 즉시 반영 필요 (이미 DDL 수정 완료)

| # | 테이블 | 변경 내용 |
|---|---|---|
| 1 | `products` | `sale_id` 컬럼 제거 |
| 2 | `chat_rooms` | `product_id` → `sale_id` 로 변경, FK 대상 `sale_events`로 변경 |
| 3 | `products` | `category_code` 타입: `VARCHAR(20)` → `category_code_enum` |
| 4 | `product_images` | `is_main` DEFAULT FALSE 추가 |
| 5 | `order_items` | `product_name` NULL → NOT NULL |
| 6 | `short_forms` | `product_id` NOT NULL → NULL |
| 7 | `cart_items` | `updated_at` 컬럼 제거 |
| 8 | `order_items` | `price` CHECK >= 0 추가 |
| 9 | ENUM 목록 | `category_code_enum ('ALL', 'TOP', 'BOTTOM', 'OUTER', 'ACCESSORY')` 추가 |

| # | 테이블 | 변경 내용 |

## 참고: Gmail SMTP 사용 방법

1. Google 계정 접속 → **보안** → **2단계 인증** 활성화
2. **앱 비밀번호** 생성 (앱: 메일, 기기: 기타)
3. 발급된 16자리 비밀번호를 `.env`에 입력:
   ```
   MAIL_USERNAME=프로젝트용이메일@gmail.com
   MAIL_PASSWORD=xxxx xxxx xxxx xxxx  (16자리 앱 비밀번호)
   ```
4. 일반 Gmail 비밀번호는 SMTP 인증에 사용 불가 — 반드시 앱 비밀번호 사용
