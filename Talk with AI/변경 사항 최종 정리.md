### 5-4. JwtTokenProvider 버그 수정

`validateToken()`이 만료된 토큰도 `true`를 반환하던 버그 수정.

```java
// 수정 전: parseClaims()가 ExpiredJwtException 잡아서 반환 → validateToken이 true 반환
// 수정 후: validateToken은 직접 파싱해서 만료 시 false 반환
public boolean validateToken(String token) {
    try {
        Jwts.parser().verifyWith(key).build().parseSignedClaims(token);
        return true;
    } catch (Exception e) {
        return false;
    }
}
// parseClaims()는 그대로 유지 (리프레시 토큰 재발급 시 만료 토큰에서 userId 추출 용도)
```

## 참고: Gmail SMTP 사용 방법

1. Google 계정 접속 → **보안** → **2단계 인증** 활성화
2. **앱 비밀번호** 생성 (앱: 메일, 기기: 기타)
3. 발급된 16자리 비밀번호를 `.env`에 입력:
   ```
   MAIL_USERNAME=프로젝트용이메일@gmail.com
   MAIL_PASSWORD=xxxx xxxx xxxx xxxx  (16자리 앱 비밀번호)
   ```
4. 일반 Gmail 비밀번호는 SMTP 인증에 사용 불가 — 반드시 앱 비밀번호 사용
